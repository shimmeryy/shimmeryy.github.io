SpringMVC，提供了构建Web程序的MVC模块。

SpringMVC

1. Introduciton

1.1 回顾MVC

MVC即Model、View、Controller的简写，是一种软件设计规范，将业务逻辑、数据层、视图层分离，降低了视图与业务逻辑的双向耦合关系。

1.2 Why SpringMVC

Spring Web MVC是基于Servlet API构建的原生Web框架，是Spring框架的一个模块，实现了MVC的轻量级Web框架。

SpringMVC的特点：

  1. 轻量级，简单易学。
  2. 基于请求响应模型的MVC框架。
  3. 功能强大：RESTful、数据验证、格式化、本地化、主题等。
  4. 约定大于配置。

1.3 DispatcherServlet

SpringMVC核心围绕DispatcherServlet设计，DispatcherServlet的作用是将请求分发到不同的处理器中。

即以请求为驱动，围绕一个中心Servlet（DispatcherServlet）分派请求及提供其他功能。

1.4 SpringMVC工作原理

  1. 用户发送请求，DispatcherServlet接受并拦截用户的请求。
  2. DispatcherServlet调用HandlerMapping（处理器映射），根据url查找与请求匹配的Handler。
  3. DispatcherServlet调用HandlerAdapter（处理器适配器），按照特定的规则去执行Handler。
  4. Handler处理业务逻辑，让具体的Controller执行，返回Model And View。
  5. DispatcherServlet调用ViewResolver（视图解析器）来解析Model And View，将解析的逻辑视图名返回给DispatcherServlet。
  6. DispatcherServlet将Model数据传递给view，调用具体的视图，从而返回给用户。

2. Hello SpringMVC

  1. 通过Maven导入Spring-webmvc的依赖。

  2. 配置web.xml：核心是注册DIspatcherServlet

    <?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">

    <!-- 注册DispatcherServlet -->
    <servlet>
        <servlet-name>springmvc</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <!-- 配置SpringMVC配置文件 -->
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:springmvc-servlet.xml</param-value>
        </init-param>
        <!--设置启动级别为1 启动即加载而非请求Servlet后才加载-->
        <load-on-startup>1</load-on-startup>
    </servlet>

    <!--/ 匹配所有的请求；（不包括.jsp）-->
    <!--/* 匹配所有的请求；（包括.jsp）-->
    <!--一般来说我们都用/ 因为用/*会导致拼接前后缀出问题-->
    <servlet-mapping>
        <servlet-name>springmvc</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>

</web-app>

  3. 配置SpringMVC配置文件

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
       http://www.springframework.org/schema/beans/spring-beans.xsd">
  
    <bean class="org.springframework.web.servlet.handler.BeanNameUrlHandlerMapping"/>

    <bean class="org.springframework.web.servlet.mvc.SimpleControllerHandlerAdapter"/>

    <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver" id="InternalResourceViewResolver">
        <!--前缀-->
        <property name="prefix" value="/WEB-INF/jsp/"/>
        <!--后缀-->
        <property name="suffix" value=".jsp"/>
    </bean>

</beans>

  4. 创建业务类HelloController：

    package cn.shimmery.controller;

import org.springframework.web.servlet.ModelAndView;
import org.springframework.web.servlet.mvc.Controller;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

public class HelloController implements Controller {

    @Override
    public ModelAndView handleRequest(HttpServletRequest httpServletRequest, HttpServletResponse httpServletResponse) throws Exception {
        ModelAndView mv = new ModelAndView();

        mv.addObject("msg","helloSpringMVC");

        mv.setViewName("hello");

        return mv;
    }
}

    业务类必须继承Controller接口，复写接口中的handleRequest方法。

    我们通过定义ModelAndView对象来设置模型和视图。通过addObject方法可以设置特定的属性名称，通过setViewName方法可以设置视图的名称，从而通过视图解析器解析到对应的视图中。（比如这里就是/WEB-INF/jsp/hello.jsp）

  5. 在SpringMVC配置文件中注册业务类的bean：

    <bean id = "/hello123" class="cn.shimmery.controller.HelloController"/>

    Hint：这里bean的id即为请求路径名，比如这里设置成/hello123，我们就可以通过localhost:8080/hello123访问这个界面，注意不要丢了/，不要乱起id。

  6. 创建对应的视图：

    <%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>Title</title>
</head>
<body>
${msg}
</body>
</html>

  7. 启动tomcat，发布。

  • 常见错误：404 找不到界面

    原因1：对应的依赖没有成功注入到发布的项目中。我们可以通过IDEA的File-Project Structure- Artifacts-Output Layout-WEB-INF 中创建lib目录，将所有的依赖jar包导入到该目录中。

    原因2：在注册业务类的bean的时候id写错了，id应为发布后的请求路径名。

    原因3：路径写错了。

    原因4：*映射路径写成了/而非/，导致视图解析器疯狂加.jsp套娃找不到界面。

3. Annotation

相比于xml配置，用注解在配置上有所不同。

首先在springmvc-servlet.xml配置中：

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xmlns:mvc="http://www.springframework.org/schema/mvc"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
       http://www.springframework.org/schema/beans/spring-beans.xsd
       http://www.springframework.org/schema/context
       https://www.springframework.org/schema/context/spring-context.xsd
       http://www.springframework.org/schema/mvc
       https://www.springframework.org/schema/mvc/spring-mvc.xsd">

    <context:component-scan base-package="cn.shimmery.controller"/>

    <mvc:default-servlet-handler />

    <mvc:annotation-driven />

    <!-- 视图解析器 -->
    <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver"
          id="internalResourceViewResolver">
        <property name="prefix" value="/WEB-INF/jsp/" />
        <property name="suffix" value=".jsp" />
    </bean>

</beans>

导入使用SpringMVC注解的环境，同时设置扫描包，通过<mvc:default-servlet-handler />过滤静态资源，通过 <mvc:annotation-driven />加载处理器映射器和处理器适配器。剩下的我们只需自己配置视图解析器即可。

controller类：

package cn.shimmery.controller;

import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
public class HelloController {

    @RequestMapping("/hello123")
    public String Hello(Model model){
        model.addAttribute("msg","helloAnnotationMVC!");
        return "hello";
    }
}

我们通过**@Controller注解将这个自定义的controller交由SpringIoC容器管理**。

通过@RequestMapping的value属性设置映射请求路径。（这个注解可以在类上加，也可以在方法上加。如果类和方法上都加了，那么访问方法对应的视图我们就要用类的请求路径+方法上的请求路径拼接起来才能访问成功）

在方法中，我们必须设置String返回值，返回值即为视图解析器调用的视图名称。

在方法中，我们传递Model类型的参数，通过model对象的addAttibute方法设置参数，将数据带到视图中。

4. Controller

控制器负责提供用户访问应用程序的行为，从上面的两个例子可以看出来，我们可以通过实现Controller接口并复写handlerRequest方法或者使用注解来实现。

如果通过实现Controller接口的方法来实现，注意需要配置bean。

一般来说，在开发中我们都使用注解来实现。

5. RESTful风格

RESTful：Representational State Transfer 是一种资源定位及资源操作的风格。

特点：

  1. 每一个URL代表一种资源。
  2. 客户端可以使用GET、POST、PUT、DELETE四种操作对服务端资源进行操作。其中GET用于获取资源，POST用于新建资源（或者是更新资源），PUT用于更新资源，DELETE用于删除资源。
  3. 客户端与服务器端之间的交互在请求之间是无状态的，从客户端到服务器端的每个请求都必须包含理解请求所必须的信息。

在传统方式，我们传参一般用?传参，方法单一，只能用POST和GET。

使用RESTful风格操作资源，我们可以通过/传参，同时同一个URL可以对应多种不同的请求方式。

例：使用RESTful风格传参数

@RequestMapping(value = "/rest/{a}/{b}",method = RequestMethod.GET )
public String rest(@PathVariable int a,@PathVariable int b, Model model){
    int result = a + b;
    model.addAttribute("msg","结果是"+result);
    return "hello";
}

我们可以在变量中加@PathVariable注解，通过路径变量来传递参数，路径变量用{}括起来。

同时，我们可以显式的指定请求方式，使用@RequestMapping中的method属性，指定GET、POST、PUT、DELETE等方式。同时也可以使用@GetMapping，@PostMapping等组合注解，相当于简化配置书写。

常见错误：

  1. 400:传递参数与实际类型不匹配，无法完成类型转换。
  2. 405:请求方式与所需的请求方式不匹配，eg：“Request method ‘GET’ not supported”

6. 结果跳转方式

6.1 ModelAndView

我们可以通过创建ModelAndView对象来传递属性和对应的视图界面。

6.2 Servlet API

我们可以通过传递HTTPServletResponse对象进行输出语句、重定向，也可以通过传递HTTPServletRequest对象进行转发。即原生的Servlet API方式。

6.3 SpringMVC实现转发和重定向

我们可以在Controller类的方法的String返回值中设置实现转发和重定向。

eg：

return "forward:/index.jsp"; 通过forward: 关键字来实现转发界面

return "redirect:/index.jsp";通过redirect: 关键字来实现重定向界面

7. 数据处理

  1. 提交的域名称和处理方法的参数名一致

提交数据 : http://localhost:8080/hello?name=zhangsan

@RequestMapping("/hello")
public String hello(String name){
   System.out.println(name);
   return "hello";
}

后台输入：zhangsan

  1. 提交的域名称和处理方法的参数名不一致

提交数据 : http://localhost:8080/hello?username=zhangsan

解决方案：通过@RequestParam(“”)来指定提交的域名称

@RequestMapping("/hello")
public String hello(@RequestParam("username") String name){
   System.out.println(name);
   return "hello";
}
  1. 提交一个对象

要求提交的表单域与对象的属性名一致，参数传递对象即可，无需做另外的设置。

eg:

实体类

@Data
public class User {
   private int id;
   private String name;
   private int age;
}

提交数据 : http://localhost:8080/hello/user?name=zhangsan&id=1&age=18

@RequestMapping("/user")
public String user(User user){
   System.out.println(user);
   return "hello";
}
