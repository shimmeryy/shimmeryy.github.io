Maven，优秀的项目管理及自动构建工具。

Maven

Chapter 1. Introduction

1.1 Concept

Maven(Apache Maven)是一个项目管理及自动构建工具，基于项目对象模型(POM Project Object Model)概念，Maven利用一个中央信息片断能管理一个项目的构建、报告和文档的步骤。

Maven包含了一个项目对象模型，一组标准集合，一个项目生命周期(Lifecycle)，一个依赖管理系统(Dependency Management System)和用来运行定义在生命周期阶段中插件(plugin)目标的逻辑。

1.2 Superiority

  1. 在大的Java工程项目中，我们往往需要导入多个jar包，手动管理这些jar包相当麻烦，而且会使得项目所占用的内存增大，同时存在jar包冲突以及版本等问题。而我们的Maven可以帮助我们管理jar包和解决jar包冲突问题（通过scope指定jar包作用域）
  2. Maven提供了生命周期中的各种方法，能够帮助我们管理jar包，编译代码，进行junit单元测试，打包成war包，部署安装并发布项目，大大简化了开发过程中的操作。

1.3 Install for MacOS

  1. 下载：

    下载地址：http://maven.apache.org/download.cgi

    Binary tar.gz archive [apache-maven-3.8.1-bin.tar.gz] 下载tar.gz文件 一般在MacOS和Linux系统中使用

  2. 配置

    1. 在终端中：vim ~/.bash_profile 通过 i 命令切换到insert模式

    2. 添加环境变量：

      MAVEN_HOME= // maven路径（可以通过把maven解压后的文件夹拖到终端中获得路径）
PATH=$MAVEN_HOME/bin:$PATH
export MAVEN_HOME
export PATH

    3. 使用 :wq 命令保存环境变量文件

    4. 使用 source .bash_profile 命令保存配置文件

    5. 输入mvn -v 有版本号信息则说明配置成功

      Maven home: /Users/shimmery/Downloads/apache-maven-3.8.1
Java version: 1.8.0_282, vendor: Azul Systems, Inc., runtime: /Library/Java/JavaVirtualMachines/zulu-8.jdk/Contents/Home/jre
Default locale: zh_CN, platform encoding: UTF-8
OS name: "mac os x", version: "11.2.3", arch: "aarch64", family: "mac"

  3. zsh下找不到maven（mvn）命令

    解决方案：

    1. vi ~/.zshrc 进入配置文件
    2. 添加source ~/.bash_profile命令使得每次进入zsh会自动加载.bash_profile文件

  4. Maven换源

    如果我们在国内使用Maven自带的网络仓库，不科学上网的话下载一个模版配置得半小时… 所以建议换成阿里的源

    步骤：

    1. 找到Maven目录下的conf/settings.xml文件

    2. 在

      <mirror>
  <id>mirrorId</id>
  <mirrorOf>repositoryId</mirrorOf>
  <name>Human Readable Name for this Mirror.</name>
  <url>http://my.repository.com/repo/path</url>
</mirror>

      后面加

      <mirror>
      <id>aliyunmaven</id>
      <mirrorOf>central</mirrorOf>
      <name>阿里云公共仓库</name>
      <url>https://maven.aliyun.com/repository/public</url>
</mirror>

  5. 完事后在终端使用mvn help:system 命令看看有没有aliyun源即可

Chapter 2. Maven Repository

2.1 Local Repository

本地仓库用来存储从远程仓库或者中央仓库下载的jar包和一些插件，当我们使用jar包和插件时候会优先从本地仓库来查找。本地仓库设置是在Maven文件夹的conf/settings.xml文件中，直接添加这一行即可

<localRepository>需要指定的maven本地仓库地址</localRepository>

2.2 Central Repository

Maven内置的中央仓库 https://search.maven.org/ ，中央仓库包含了绝大多数流行的开源Java构件，以及源码、作者信息、SCM、信息、许可证信息等。一般来说，简单的Java项目依赖的构件都可以在这里下载到。Maven的中央仓库需要联网才能访问到。当在本地仓库找不到时会在中央仓库找jar包。

2.3 Remote Repository

常用远端仓库：https://mvnrepository.com/

里面有大量的jar包，使用时候直接配置pom.xml，之后load maven即可 eg：

<!-- https://mvnrepository.com/artifact/org.springframework/spring-jdbc -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-jdbc</artifactId>
    <version>5.3.6</version>
</dependency>

Maven依赖搜索的顺序

  1. Maven会先在本地仓库找，若找不到则会到中央仓库找依赖文件。
  2. 如果设置了远端仓库则会在远端仓库中找依赖文件。
  3. 如果都找不到Maven会抛出错误。

Chapter 3. Maven Catalogue

3.1 Main&Test

src资源路径下分为main和test两个文件夹，其中main中主要放核心代码，在java文件夹中放.java文件，在resources文件夹中放项目的资源文件比如配置文件。在test文件夹中主要放测试代码，在java文件夹中放测试的.java文件，在resources文件夹中放测试的资源文件。

3.2 Target

在执行Maven命令的时候会自动创建target文件夹，存放编译后生成的文件。

Chapter 4. Maven Command

4.1 compile

compile是Maven的编译命令，将src/main/java文件夹下的.java文件编译成字节码文件输出到新生成的targer文件夹中。

4.2 test

test是Maven的测试命令，先执行compile，再将src/test/java文件夹下的.java文件编译成字节码文件输出到新生成的targer文件夹中。

4.3 clean

clean命令可以清除之前Maven进行的操作，删除target目录。

4.4 package

package是Maven的打包命令，在打包前先执行compile和test命令，之后对于Java工程打包成jar包，对于Web工程打包为war包。

4.5 install

install是Maven的安装命令，先执行compile-test-package命令，将jar包或war包发布到本地仓库。

4.6 deploy

通过deploy命令可以将maven的web项目部署到远端服务器。

4.7 LifeCycle

为了完成Default生命周期，这些阶段会被按顺序的执行。（即：比如我们使用package命令，实际上我们会执行validate-compile-test-package这些所有的命令。

在Maven中主要有三套标准的生命周期：

  • clean：项目清理的处理
  • **default(或 build)**：项目部署的处理
  • site：项目站点文档创建的处理
