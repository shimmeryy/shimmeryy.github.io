设计模式-代理模式

代理模式

1. 静态代理

在一般的模式下，我们是客户端与真实对象直接进行通讯。但这种模式可能会存在一定的弊端，比如我们想要扩展真实对象，则需要对真实对象的代码进行大规模的改动，这显然不符合我们的设计原则。

从而我们引入了代理模式，即在真实对象的基础上引入其代理对象，代理对象与真实对象同时实现抽象对象的接口，代理对象通过重写接口中的方法来实现扩展功能。

例：房东通过中介租房

Rent接口:

package cn.shimmery.pojo;

public interface Rent {
    public void rent();
}

Host真实对象类:

package cn.shimmery.pojo;

public class Host implements Rent{
    @Override
    public void rent() {
        System.out.println("房东在出租房子");
    }
}

Proxy代理对象类:

package cn.shimmery.pojo;

public class Proxy implements Rent {
    public Host host;

    public void setHost(Host host) {
        this.host = host;
    }

    public Proxy(Host host) {
        this.host = host;
    }

    @Override
    public void rent() {
        System.out.println("中介带你看房子");
        host.rent();
        System.out.println("中介跟你签约");
    }
}

Client 客户端类:

package cn.shimmery.pojo;

public class Client {
    public static void main(String[] args) {
        Host host = new Host();
        Proxy proxy = new Proxy(host);
        proxy.rent();
    }
}

我们通过静态代理实现了每个类功能的进一步分割，解耦。同时可扩展性更强，我们仅需通过修改代理类的代码即可实现对真实对象类业务逻辑的扩展。

但是通过静态代理模式，我们每多一个真实对象类，就要写这个真实对象类对应的代理类，会导致代码量过大，影响开发效率，同时难以维护。

2. 动态代理

相对于静态代理，动态代理强调的是“动态”这两个字，即在动态代理中，代理类都是自动生成的，无需我们自己去写，减少了代码的工作量。

动态代理一般来说主要分为两种，第一种是基于接口的动态代理，第二种是基于类的动态代理。

  • 基于接口的动态代理：JDK动态代理
  • 基于类的动态代理：cglib动态代理
  • 同时javassist也是一种常用的动态代理

关于JDK动态代理，涉及到两个类。

public interface InvocationHandler

InvocationHandler is the interface implemented by the invocation handler of a proxy instance.

Each proxy instance has an associated invocation handler. When a method is invoked on a proxy instance, the method invocation is encoded and dispatched to the invoke method of its invocation handler.

在这个类中仅有一个方法：

Object invoke(Object proxy, Method method, Object[] args)Processes a method invocation on a proxy instance and returns the result

通过调用这个invoke方法即可执行代理中的方法。

public class Proxy	extends Object	implements Serializable

Proxy provides static methods for creating dynamic proxy classes and instances, and it is also the superclass of all dynamic proxy classes created by those methods.

在Proxy类中，最重要的方法：

public static Object newProxyInstance(ClassLoader loader,
                                      Class<?>[] interfaces,
                                      InvocationHandler h)
                               throws IllegalArgumentException

Returns an instance of a proxy class for the specified interfaces that dispatches method invocations to the specified invocation handler.

  • Parameters:

    loader - the class loader to define the proxy class 一般是代理类的getClass.getClassLoader

    interfaces - the list of interfaces for the proxy class to implement 代理类需要实现的接口 .getClass.getInterfaces

    h - the invocation handler to dispatch method invocations to 一个InvocationHandler

使用：依旧以房东租房为例子

Rent接口:

package cn.shimmery.pojo;

public interface Rent {
    public void rent();
}

Host类:

package cn.shimmery.pojo;

public class Host implements Rent{
    public void rent(){
        System.out.println("房东来租房子啦");
    }
}

DynamicProxy动态代理实现类:

package cn.shimmery.pojo;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

public class DynamicProxy implements InvocationHandler {

    public Rent rent; // 在动态代理类中加载接口

    public DynamicProxy(Rent rent) {
        this.rent = rent;
    }

  	// 获取动态代理的实例
    public Object getProxy(){
        return Proxy.newProxyInstance(this.getClass().getClassLoader(),rent.getClass().getInterfaces(),this);
    }
  	// 通过重写invoke方法来实现对业务逻辑的扩展
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        before();
        Object result = method.invoke(rent, args);
        after();
        return result;
    }
    public void before(){
        System.out.println("中介带你看房子");
    }
    public void after(){
        System.out.println("中介与你签约");
    }
}

Client客户端类:

package cn.shimmery.pojo;

public class client {
    public static void main(String[] args) {
        Host host = new Host(); // 先创建真实被代理的对象
        DynamicProxy dp = new DynamicProxy(host); //通过构造器注入被代理的对象
        Rent proxy = (Rent) dp.getProxy(); //获取动态代理对象并强制类型转换
        proxy.rent(); // 执行代理对象中的方法
    }

}

更进一步的扩展动态代理类：

package cn.shimmery.pojo;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

public class DynamicProxy implements InvocationHandler {

    public Object object; // 代理类需要实现的接口

    public DynamicProxy(Object object) {
        this.object = object; // 通过构造器注入接口
    }

    public Object getProxy(){
        return Proxy.newProxyInstance(this.getClass().getClassLoader(),object.getClass().getInterfaces(),this);
    } // 获取动态代理对象的实例
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        before();
        Object result = method.invoke(object, args);
        after();
        return result;
    } // 实现动态代理对象业务逻辑的扩展
    public void before(){
    } // 扩展
    public void after(){
    } // 扩展
}

动态代理的好处：

  1. 代理逻辑与业务逻辑相互独立，没有强耦合关系。
  2. 实现非入侵式的代码扩展。
