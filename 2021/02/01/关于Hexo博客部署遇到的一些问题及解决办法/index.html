关于Hexo博客部署遇到的一些问题及解决办法

Spawn failed

  • 使用hexo d后遇到了这种情况

经过了万能的百度后找到了解决办法

这种情况出现的原因是博客之前进行了多余的本地提交，导致跟git仓库中的不同步；

在.deploy_git\.git\logs（.git文件夹可能被隐藏，强制进入）中的HEAD文件（用万能的VSCode文本查看器）

对比github仓库里的最后提交时间

发现多了一个多余的本地提交，必须先返回到原来的状态才可

解决问题代码:

git reset --hard commitId

这里的commitId是HEAD中的每一行的第二段ID,找到最后提交的那个状态返回到该状态即可解决该问题;

文章目录收缩错误

这里因为忘了截图所以没有图…

大概的问题是当时我的文章(markdown格式)用的是

h1
h3
h4
h3
h4

这时目录把我的第一个h3识别成了h2,把我的第一个h4识别成了h3,导致目录收缩错误,变得奇奇怪怪的;

经过查询后发现,原因可能是这种目录的检索最好是根据

h1
h2
h3
h4
h5
h6

这样有层次来的,越级可能会导致识别出错(之前有时候是没问题的);当我把第一个h1改成了h2后解决了问题;

文章目录查询错误

当我点这个5.4的时候跳到了Chapter6 有点奇奇怪怪的

经过了控制变量等操作 发现了可能的问题所在 那就是在使用##设置标题的时候 标题前面一行是```java的结束```标签 可能是跟#组合产生了一些问题?

单独看文章展示效果 这里的5.2和5.3都没有对齐 说明肯定存在问题 之后我在```后加了个—分隔符 或者是再写文字 即可解决类似问题

不过比较奇怪的是并不是所有的这样都会导致问题 之前写的markdown就没有问题 不太清楚具体问题产生的来源 待我之后再看看这个目录解析的源码…
